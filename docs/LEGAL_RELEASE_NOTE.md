# Legal release note

The repository currently ships a BSD-3-Clause license file as a **provisional publication choice**.

Before making the repository public, confirm which party actually owns the rights to the code and release artifacts:

- the individual authors,
- the university or research institution,
- an employer or industrial partner, or
- a joint ownership arrangement.

## Conservative rule
If ownership is not yet confirmed, keep the repository private and resolve authorship / institutional rights first. A wrong public license is harder to undo cleanly than a delayed publication.

## If the authors personally own the code
BSD-3-Clause is a reasonable scientific-software choice and is already consistent with the current repository metadata.

## If institutional ownership applies
Update the copyright line in `LICENSE`, and if needed also update:

- `pyproject.toml`
- `CITATION.cff`
- repository hosting organization / namespace

## Practical publication check
Before making the repo public, explicitly answer these questions in writing:

1. Who owns the source code?
2. Who owns the manuscript-facing aggregated results and tables?
3. Is EMBER redistribution restricted in any way by your release plan?
4. Is BSD-3-Clause approved by the actual rights holder?
