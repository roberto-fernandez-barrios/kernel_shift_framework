# Legal release note

This repository currently ships a `BSD-3-Clause` license file.

That is a reasonable scientific-software choice **provided that the actual rights holder has approved it**.

## Ownership question to resolve explicitly

Before treating the repository as the definitive public software release, confirm which party owns the rights to the code and release artifacts:

- the individual authors,
- the university or research institution,
- an employer or industrial partner,
- or a joint ownership arrangement.

## Conservative rule

If ownership is not yet confirmed, keep the public licensing position conservative and resolve authorship / institutional rights first.

A delayed clarification is easier to fix than a wrongly granted public license.

## If the authors personally own the code

`BSD-3-Clause` is a defensible and common choice for research software and is consistent with the current repository presentation.

## If institutional ownership applies

Update the relevant metadata so that the public release reflects the actual rights holder. Depending on the case, that may include:

- `LICENSE`
- `CITATION.cff`
- `pyproject.toml`
- repository namespace / hosting organization
- release notes or archival metadata

## Data and artifact scope

This repository should not be interpreted as granting rights beyond the released repository contents themselves.

In particular:

- EMBER dataset redistribution is not implied by the software license,
- third-party datasets, papers, or external tools retain their own licensing terms,
- readers must obtain EMBER and any other external dependencies according to their original terms.

## Practical publication check

Before considering the repository legally finalized, answer these questions explicitly:

1. Who owns the source code?
2. Who owns the manuscript-facing aggregated results and tables included in the repository?
3. Does the release plan respect EMBER redistribution boundaries?
4. Is `BSD-3-Clause` approved by the actual rights holder?

## Practical conclusion

This repository is suitable for public release under `BSD-3-Clause` **if and only if** that license matches the confirmed ownership situation.

If ownership has already been confirmed, this note can be read as a record of due diligence rather than as a blocker.
